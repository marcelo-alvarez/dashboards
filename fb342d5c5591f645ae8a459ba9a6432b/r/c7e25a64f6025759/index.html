Source of the diffsky sSFR offset — factor isolation (z = 1.4)

roman-covariance · perlmutter-marcelos-claude-code · on-data execution of the debug plan

Question

Report #6 found the diffsky sSFR–M\* relation ~0.45 dex below the observed/Speagle+14 main

sequence while the GSMF and SMHM match. Candidate causes (per context/diffsky-ssfr-debug.md):

little-h mass units, halo-mass definition, wrong cosmology, instantaneous-vs-averaged SFR, or

the diffstarpop model itself. This report toggles each factor on real runs and measures it.

Method

Synthetic central-halo population (N=60,000, logMh uniform 10.8–14.4) through the **same engine

the pipeline uses** (rgrspit_diffsky.mc_galpop.mc_galpop_synthetic_subs, production conda env),

centrals only. It reproduces the report-#6 signal exactly: median logsSFR(M\*=10¹⁰) = −9.225

(real m4943 box: −9.24).

Result — it is the model, not the pipeline

Every candidate fix moves median sSFR(M\*=10¹⁰) by ≤0.008 dex; the five curves are

indistinguishable. The full ~0.43 dex deficit is the **diffstarpop default main sequence at

z=1.4**.

toggleshift in median logsSFR@10¹⁰
halo mass ±0.172 dex (little-h)≤0.006 dex — no effect
c000 cosmology vs PLANCK15+0.003 dex — no effect
100-Myr-averaged vs instantaneous SFR−0.005 dex — no effect
remaining (diffstarpop model)−0.43 dex

A global or mass-dependent mass relabeling slides each halo along a fixed sSFR–M\* locus, so it

cannot change median sSFR at fixed M\* — confirmed by the ±0.172 dex toggles. (Also: the

little-h error *under*-states mass, not over-states; the sign in the debug note is reversed.

Fix it for the GSMF/SMHM mass scale, but it will not raise sSFR.)

Why the model is low

The star-forming centrals (M\*≲10¹⁰) have monotonically rising SFHs at t_obs — so the

instantaneous SFR already ≈ the 100-Myr average (no timescale fix to be had) — and form one

narrow sequence sitting uniformly ~0.43 dex below Speagle, with no distinct quenched mode. Genuine

quenching appears only at M\*≳10¹⁰·⁵ (SFHs that peak ~3 Gyr and decline), producing the sharp

high-mass downturn. The offset lives in the MAH→SFH model layer (diffmahpop+diffstarpop defaults,

as driven by mc_galpop), which is conditioned on the z=1.4 halo mass only (the Abacus halos' real

MAHs are never used).
